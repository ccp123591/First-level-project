## 公式 1：关节角度公式

### 论文里它长什么样

公式 (1) 表达的意思（论文 P47–P48 文字描述）：
> θ 表示关节角度，a、b、c 分别表示构成关节角的三个关键点坐标，**atan2** 函数用于将关键点之间的向量关系转换为关节角度值。

数学上的标准写法：

```
       ⎛  | (a − b) × (c − b) |  ⎞     180
θ = atan2 ⎜ ──────────────────── ⎟  ×  ───
       ⎝     (a − b) · (c − b)    ⎠      π
```

或者写成更直观的形式：

```
设 u = a − b（从 b 指向 a 的向量）
设 v = c − b（从 b 指向 c 的向量）
则 θ = atan2( |u × v| , u · v ) × 180/π
```

### 它在算什么

> 三个关键点 a、b、c，**b 是顶点**。算的就是 b 这个关节弯了多少度。
>
> 举例：你算膝关节角度时，a = 髋、b = 膝、c = 踝。膝盖弯曲时角度小，伸直时接近 180°。

### 各符号含义（一句话一个）

| 符号 | 是啥 | 例子 |
|---|---|---|
| **θ** | 关节角度（单位：度） | 膝盖弯到 90° 就 θ ≈ 90 |
| **a, b, c** | 三个关键点的 (x, y) 坐标 | 髋(x₁,y₁)、膝(x₂,y₂)、踝(x₃,y₃) |
| **u = a − b** | 从顶点 b 指向 a 的向量 | 膝→髋的方向 |
| **v = c − b** | 从顶点 b 指向 c 的向量 | 膝→踝的方向 |
| **u · v** | **点积** = u.x·v.x + u.y·v.y | 表示两向量"同向程度" |
| **u × v** | **叉积**（2D 下是个标量）= u.x·v.y − u.y·v.x | 表示两向量"垂直程度" |
| **\|u × v\|** | 叉积的绝对值 | 因为我们只关心夹角大小，不关心顺时针/逆时针 |
| **atan2** | 一个比 arctan 更稳的反三角函数 | 见下面解释 |
| **× 180/π** | 弧度转度 | atan2 返回弧度，乘 180/π 变成度 |

### 实际代码（`exercise.js` 第 17–23 行）

```javascript
function angleDeg(a, b, c) {
  const ab = { x: a.x - b.x, y: a.y - b.y };  // u = a - b
  const cb = { x: c.x - b.x, y: c.y - b.y };  // v = c - b
  const dot = ab.x * cb.x + ab.y * cb.y;       // u · v
  const cross = ab.x * cb.y - ab.y * cb.x;     // u × v
  return Math.atan2(Math.abs(cross), dot) * (180 / Math.PI);
}
```

——10 行代码就够了，跟论文公式 (1) 完全对应。

### 为什么用 atan2（关键加分点！）

**最常见的算夹角写法**是这个：

```
θ = arccos( u·v / (|u|·|v|) )
```

但这个写法有两个**数值稳定性**问题：
1. 当 u 和 v 几乎平行时（接近 0° 或 180°），分子分母都很接近，**除法误差被放大**
2. 由于浮点误差，`u·v / (|u|·|v|)` 可能会**轻微大于 1 或小于 −1**，丢给 arccos 直接 NaN

**atan2 的写法绕开了这两个坑**：
```
θ = atan2( |u×v| , u·v )
```
- 直接用叉积和点积构造比值，**不需要算模长**
- atan2 在整个 360° 范围内都数值稳定

**所以如果评委问**："为什么不用 arccos？"

> "用 atan2 是为了**数值稳定性**。arccos 在角度接近 0° 或 180° 时容易出现浮点误差被放大、甚至因为 |cosθ| 略超过 1 而 NaN 的问题。atan2 用叉积/点积构造比值，避开了这两个坑，整个 0~180° 范围都数值稳定。"

这是个很专业的回答，能直接给你加分。

### 为什么用 |u × v|（绝对值）

> 因为在 2D 平面里，叉积有方向性（顺时针为负、逆时针为正）。我们只关心**夹角的大小**（0~180°），不关心方向，所以取绝对值。

### 被追问怎么答

**Q1：为什么是三个关键点不是两个？**

> 关节角度天然就是三个点定义的——一个顶点 + 两条骨头。两个点只能定义一段距离，没法定义角度。

**Q2：那 z 坐标呢？BlazePose 不是给三维坐标吗？**

> BlazePose 确实输出 (x, y, z, visibility)，但本系统**只用 x、y 做平面投影**计算。理由是：① 老年居家场景里，摄像头多是正面或侧面单角度，z 估计噪声较大；② 用 2D 投影已经足够区分动作状态（如蹲下 vs 站立），更稳定。

**Q3：精度够吗？**

> 对于"动作幅度判定"这个任务足够。我们不是要算医学级别的关节角，而是判断"蹲到位了没"——只要能区分 90° vs 160° 这类大幅度差异就行。代码里还做了**5 帧滑动平均平滑**，进一步抑制单帧抖动。

**Q4：可见度（visibility）有过滤吗？**

> 有。`extractAngles()` 里检查三个关键点的 `visibility > 0.5`，否则直接返回 null（这一帧丢弃）。这避免了人体被遮挡时强行算出错误角度。

## 公式 2：综合评分公式

### 论文里它长什么样

论文 P55–P56 文字描述：
> S 为综合评分，R 为节奏分，T 为稳定分，D 为深度分，Y 为对称分，C 为完成率分。上述权重为结合训练反馈目标设定的经验性参数，其中节奏和稳定性权重较高，深度次之，对称性与完成率作为辅助指标。

数学写法：

```
S = 0.25 R + 0.25 T + 0.20 D + 0.15 Y + 0.15 C
```

| 符号 | 含义 | 单位 |
|---|---|---|
| **S** | 综合评分 | 0–100 |
| **R** | 节奏分（Rhythm） | 0–100 |
| **T** | 稳定分（sTability） | 0–100 |
| **D** | 深度分（Depth） | 0–100 |
| **Y** | 对称分（symmetrY） | 0–100 |
| **C** | 完成率分（Completion） | 0–100 |
| 0.25 / 0.25 / 0.20 / 0.15 / 0.15 | 五个权重，**和为 1.00** | — |

### 代码对应（`exercise.js` 第 210 行）

```javascript
const overall = Math.round(
    rhythm * 0.25
  + stability * 0.25
  + depth * 0.20
  + symmetry * 0.15
  + completion * 0.15
);
```

——一行加法。代码和公式 100% 对应。

### 权重为什么这样设？（必背）

| 权重 | 维度 | 为什么这个分量 |
|---|---|---|
| 25% | **节奏** | 老年人节奏不稳容易岔气、心率激增，是安全核心指标 |
| 25% | **稳定** | 动作底部抖动 = 关节负担、跌倒风险，安全核心 |
| 20% | **深度** | 训练效果指标，重要但比安全次一级 |
| 15% | **对称** | 辅助指标，长期偏侧用力会有伤病 |
| 15% | **完成率** | 辅助指标，做完比做完美次要 |

**一句话总结**：
> "权重设计**安全 > 效果 > 辅助**——节奏和稳定关乎老人安全所以最高，深度关乎训练效果次之，对称和完成率是辅助。"

## 5 个分项分各自怎么算

公式 2 只是把它们加起来。每个分项分自己的算法在论文里没展开，但你应该会一句话讲。

### R · 节奏分（rhythm）

**思路**：
- 系统设了一个目标节拍 BPM（每分钟节拍数），节拍间隔 = 60000 / BPM 毫秒
- 每完成一次动作记一个时间戳
- 算每两次完成时间差 vs 目标间隔的"绝对误差"，再取平均（MAE）
- 误差越小，分越高

**公式**：
```
R = 100 × (1 − MAE / target)，下限 0
```
其中 target = 60000 / BPM。

**举例**：
> BPM = 60，目标间隔 = 1000ms。如果你每次都正好 1000ms 完成一次，MAE = 0，R = 100。如果平均偏离 200ms，R = 100 × (1 − 200/1000) = 80。

### T · 稳定分（stability）

**思路**：
- 取动作"底部"附近的角度样本（threshold_down ± 20° 内的所有帧）
- 算这些样本的**标准差** σ（标准差越大，说明动作越抖）
- 标准差越小，分越高

**公式**：
```
T = 100 × (1 − σ / 20)，下限 0
```

**举例**：
> 你蹲到底时角度始终在 88°~92° 之间，σ ≈ 1.5，T ≈ 92。如果你蹲到底但抖得 75°~105° 都有，σ ≈ 10，T ≈ 50。

### D · 深度分（depth）

**思路**：
- 每次完成一次动作，记录这一次的**最低角度**（最深位置）
- 求所有次动作最低角度的平均值
- 跟"理想深度"比较，偏差越小越好
- 理想深度 = max(45°, threshold_down − 10°)

**公式**：
```
D = 100 − |mean(min_angles) − ideal_depth| × 2.5，下限 0
```

**举例**：
> 深蹲 threshold_down = 90°，ideal_depth = 80°。你每次蹲到平均 85°，偏差 5°，D = 100 − 5×2.5 = 87.5。如果只蹲到 100°，偏差 20°，D = 100 − 50 = 50。

### Y · 对称分（symmetry）

**思路**：
- 同时记录每一帧的左侧关节角和右侧关节角
- 算所有帧 `|左角 − 右角|` 的平均值（MAE）
- 偏差越小，分越高

**公式**：
```
Y = 100 − MAE × 2，下限 0
```

**举例**：
> 你每帧左右角度差平均 5°，Y = 100 − 10 = 90。差 25°，Y = 50。

### C · 完成率分（completion）

**思路**：完成的次数除以目标次数。

**公式**：
```
C = min(100, reps / target_reps × 100)
```

**举例**：
> 目标 10 次，做了 8 次，C = 80。做了 12 次（超额），C = 100（封顶）。

## 被追问怎么答（必背 Q&A）

### Q1：5 个权重 0.25/0.25/0.20/0.15/0.15 怎么定的？

> "经验性参数。**安全维度 > 效果维度 > 辅助维度**：节奏和稳定关乎老年人安全所以各 25%；深度关乎训练效果给 20%；对称和完成率作为辅助各 15%。论文 P56 也说明这些是'经验性参数'，'后续可根据测试结果进一步调整'——意思是这套权重不是一锤定死，是当前认知下的合理初值。"

### Q2：为什么不用机器学习自动学权重？

> "三个原因：① 这是**规则驱动 + 可解释**的工程化设计，老年用户能理解'我哪一项扣分'，机器学习黑盒做不到；② 没有大量标注数据；③ 课程论文定位是系统设计与实现，不是算法研究——见论文 P81 第二段。"

### Q3：分项分跟综合分怎么呈现给用户？

> "训练报告页同时展示：综合分 + 5 个雷达图维度。这样老人看综合分知道整体水平，看雷达图知道**哪一项是短板**——这就是论文反复强调的'可解释性'。"

### Q4：稳定分用 σ/20 这个 20 是什么？

> "经验值。在阈值附近 ±20° 内的角度方差，σ = 20° 时基本认为'完全不稳'，给 0 分；σ = 0° 时给 100 分。线性映射。后续可以根据真实老年用户实测进一步标定。"

### Q5：节奏分用 mae/target 不是会偏严吗？

> "确实，BPM = 60 时，每次偏 500ms 节奏分就只剩 50。这是有意设计的——老年人节奏不稳是**安全风险**（心率波动），所以节奏分给得严一点，权重也最高。"

### Q6：R / T / D / Y / C 这套维度是你提出的还是参考来源的？

> "维度划分参考运动训练通用评估视角（节奏、稳定、幅度、对称、完成度），不是某一篇文献的原创复现，但在**可解释评分应用于老年居家场景**这个组合上是本文的工程贡献。论文创新点 ③ 写的就是这个。"

### Q7：评分有跟"金标准"对比吗？

> "**没有**。论文 5.4 节第二点'规则驱动评分能提供基础训练反馈但不能替代专业康复指导'已经坦诚说明。后续工作（论文 6.2 节）计划引入真实老年用户测试和专业康复师标注做对照。"

> ⚠️ 这条是个潜在被攻击点——评委会想问"你怎么证明你的评分是对的"。**承认是当下不足、列入未来工作**就是最稳的答法。
