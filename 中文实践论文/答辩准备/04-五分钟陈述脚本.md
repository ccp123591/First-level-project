## 【0:30–1:15】研究背景与动机（45 秒）

> 首先是**研究背景**。// [切到图 1：老年居家健身场景需求示意图]
>
> 随着人口老龄化加深 / 老年人居家锻炼需求持续增长。/ **世界卫生组织 2020 年发布的身体活动指南**指出 / 规律运动能显著改善老年人**心肺、肌肉、平衡**等功能 / 是促进健康老龄化的重要方式。/ 同时**多项综述研究**也证实 / 科学训练能有效**降低老年人跌倒发生率**。
>
> 但是 / 真实居家场景里 / 老年人**独立锻炼**面临几个问题：/ 动作做不到位、节奏把握不稳、缺现场指导、对复杂数字产品操作不熟。// 现有的健身视频平台或打卡 App / 大多只是**放视频和记次数** / **没有实时的动作纠错和反馈能力**。/ 所以我希望构建一套 / **门槛低、可纠错、能离线用、对老年人友好**的居家训练辅助系统。

## 【1:15–2:30】总体设计与技术选型（75 秒）

> 这就是我做 FitCoach 的初衷。// [切到图 2：系统总体架构图]
>
> 系统采用**前后端分离**架构 / 分三层：
>
> **第一层是浏览器端训练层**。/ 它**承担核心训练能力**——/ 包括摄像头采集、姿态识别、动作判定、计数、评分、本地保存 / 这些环节**全部在浏览器端本地完成**。/ 这意味着**视频数据不出用户本机** / 既保护隐私 / 又能在弱网环境下保持基本可用。
>
> **第二层是后端服务层** / 用 **Spring Boot** 实现 / 主要负责用户认证、训练记录管理和扩展业务模块的接口支撑。/ **它不介入每一次的实时识别**——这一点是有意识的设计——/ 把核心训练流程跟网络解耦。
>
> **第三层是数据存储层** / 采用**双层存储**：/ 浏览器端的 **IndexedDB** 负责本地训练记录 / 保证离线可用；/ 后端数据库负责长期记录管理和扩展业务。
>
> 这套架构的核心思想是**本地优先** / 训练核心**不依赖网络** / 这是论文的重要设计选择 / 也是**面向老年居家场景的关键适配**。

## 【2:30–3:45】核心模块设计与实现（75 秒）

> 接下来是**核心模块的设计与实现**。// [切到图 3：训练主链路流程图]
>
> 训练主链路是这样：/ 用户选择动作和目标次数 → 浏览器初始化**MediaPipe Pose**模型 → 摄像头采帧 → 模型输出**33 个人体关键点** → 进入动作判定逻辑。
>
> 动作判定我做了三件事：/
>
> **第一 / 计算关节角**。/ [PPT 切到公式 1] / 用三个关键点构造夹角 / 公式里用了 **atan2 函数**而不是 arccos / 这是为了**数值稳定性**——/ atan2 在角度接近 0 度或 180 度时不会因浮点误差出现 NaN。
>
> **第二 / 状态机计数**。/ 设两个阈值——/ 角度低于"下限"算"下" / 高于"上限"算"上"——/ **必须先下后上才计数 +1**。/ 这避免了瞬间多次误判。
>
> **第三 / 五维评分**。/ [PPT 切到公式 2] / 综合分由**节奏、稳定、深度、对称、完成率**五个维度加权得到 / 权重设计原则是 / **安全大于效果大于辅助** —— /
> **节奏和稳定各 25%**——这是老人安全核心 / **深度 20%** —— 训练效果 / **对称和完成率各 15%** —— 辅助。/ **不只给一个总分** / 还给五个维度的子分 / 让老人**知道哪一项是短板**——/ 这就是论文反复强调的**可解释性**。

### 站上去之后

- **先扫一眼评委**——眼神接触一下，不用盯

- **不要看 PPT 念**——你的眼神要在评委和 PPT 之间切换 60/40

### 念稿子的人怎么显得不像在念稿子

1. **每段第一句完全脱稿**——背下来
2. **关键术语脱稿** ——比如"MediaPipe Pose"、"IndexedDB"念到时抬头
3. **数字脱稿** ——5 个维度、25%、33 个关键点……
4. **"我"和"我们"" — 你"" — 各位老师"** 这些指代词，每次说完都抬头
5. **公式那段**用手指 PPT，眼睛跟手——自然抬头
