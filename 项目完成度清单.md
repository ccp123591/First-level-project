# FitCoach 项目完成度清单

本清单基于当前仓库代码整理，判断标准以“是否已经形成真实可用闭环”为主。

- 主评估对象：`home-fitness-fullstack/`
- 参考对象：`home-fitness-pwa/`

---

## 一、已完成

### 1. 工程骨架与基础设施

- 前后端工程结构完整，可分别启动运行。
- 已提供 `docker-compose.yml`，可同时拉起前后端。
- 后端已完成 JWT 安全配置、Swagger、H2 / MySQL 环境配置。
- 已提供开发种子数据，包括动作库、官方训练计划、徽章、管理员/演示账号。
- 前端已注册 Service Worker，具备 PWA 基础离线能力。

### 2. 训练核心链路

- 已实现摄像头采集与 MediaPipe Pose 姿态识别。
- 已实现动作计数、角度判断、多维评分。
- 已实现语音提示、节拍器、自动暂停。
- 已实现训练完成弹窗与成绩展示。

### 3. 本地数据闭环

- 训练记录可写入 IndexedDB。
- 已实现本地记录列表页。
- 已实现记录详情页。
- 已实现趋势图、热力图、CSV 导出。
- 已实现本地同步状态展示。

### 4. 设置系统

- 已实现动作阈值设置。
- 已实现 BPM、TTS 语速、主题、每周目标设置。
- 已实现 AI 教练开关、自动暂停开关。
- 设置支持本地持久化保存。

### 5. 分享能力

- 已实现训练成果海报生成。
- 已支持导出 PNG 海报。

### 6. 后端认证基础

- 已实现邮箱登录。
- 已实现注册。
- 已实现游客登录。
- 已实现刷新 Token。
- 已实现获取当前用户信息。
- 已实现登出接口。
- 已提供管理员和演示账号种子数据。

### 7. 后端训练记录能力

- 已实现训练记录创建。
- 已实现分页查询、详情查询。
- 已实现备注更新、删除。
- 已实现批量同步接口。
- 已实现 CSV 导出接口。

### 8. 后端用户基础能力

- 已实现用户资料读取与更新。
- 已实现头像上传。
- 已实现个人统计。
- 已实现训练日历数据。
- 已实现关注 / 取关。
- 已实现粉丝 / 关注列表。

### 9. 后端内容与分析能力

- 已实现动作库 CRUD。
- 已实现徽章解锁判定逻辑。
- 已实现周榜 / 月榜 / 好友榜接口。
- 已实现管理后台基础接口，包括 Dashboard、用户管理、训练记录管理、数据分析。

## 二、半完成

### 1. 登录注册产品闭环

- 后端鉴权接口已实现。
- 但前端 `Login` 页面仍使用 `mock-token`，没有真正接入 `authApi`。
- 现状属于“后端可用、前端未接完”。

### 2. AI 教练

- 前端训练报告中已经接入 AI 教练组件。
- 前端会请求 `/api/coach/feedback`，失败时还有本地兜底点评。
- 但后端当前返回的是 mock 文案，不是真正调用 Claude。
- 现状属于“前端已接、后端能力未落地”。

### 3. 离线同步

- 已有本地 `synced` 标记。
- 已有 `getUnsynced()` 和后端批量同步接口。
- 但当前没有看到完整的“登录后 / 恢复联网后自动补传”调度流程。
- 现状属于“同步基础具备，自动化补传未闭环”。

### 4. 训练计划

- 后端已实现计划列表、详情、采用、放弃、进度更新等接口。
- 前端 `Plans` 页面仍主要使用本地常量数据。
- 社区计划市场仍是占位。

### 5. 社交动态

- 后端已实现 Feed、发帖、点赞、评论。
- 前端 `Feed` 页面当前仍使用本地假数据。
- 现状属于“后端有能力，前端尚未对接”。

### 6. 排行榜

- 后端周榜、月榜、好友榜接口已实现。
- 前端 `Leaderboard` 页面当前仍使用 `mockData`。
- 好友榜界面仍为占位。

### 7. 个人中心 / 徽章

- 前端 `Profile` 目前主要基于本地记录计算统计。
- 徽章展示和解锁判断也主要使用前端本地数组逻辑。
- 后端统计与徽章接口虽然存在，但前端未真正接上。

### 8. 管理后台

- 后端管理后台接口已实现一批核心能力。
- 前端 `Admin` 页面仍是示意数据与占位 UI。
- 现状属于“后端可用、前端未接完”。

### 9. 云端记录浏览

- 后端已提供训练记录列表、详情、导出等接口。
- 前端 `Records` 和 `RecordDetail` 目前主要读取本地 IndexedDB。
- 尚未形成完整的云端记录浏览闭环。

- 短信验证码发送接口为占位。

- “加入挑战”不做持久化。
- 挑战排行榜接口当前返回空列表。

### 4. 社区计划市场前端

- 页面仍明确显示“即将上线”。
- 尚未与后端计划市场接口打通。

### 5. 自动化测试体系

- 前端没有测试配置。
- 仓库中也看不到成型的后端测试集。
- 当前整体更偏展示型原型，而非测试完备的生产工程。

## 四、补充说明

### 1. 关于 `home-fitness-pwa`

`home-fitness-pwa/` 可以视为旧版纯前端原型，主要覆盖以下能力：

- 训练
- 本地记录
- 离线能力
- PWA 安装与运行

它不包含以下完整能力：

- 账号体系
- 后端服务
- 社交模块
- 管理后台

### 2. 关于根目录其他文件

根目录下的 `.docx` 和截图文件更像比赛提交材料、演示素材或验收截图，不属于运行时功能模块。

### 3. 项目整体判断

当前仓库更接近一个“比赛 / 课程展示型的完整产品原型”：

- 核心卖点“AI 健身陪练 + 姿态识别 + 本地记录 + PWA 离线”已经跑通。
- 账号体系、AI 真接入、社交生态、计划市场、管理后台前端仍未完全收口。
